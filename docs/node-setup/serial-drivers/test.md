Этот раздел помогает убедиться, что:

- USB‑кабель действительно передаёт данные, а не только заряжает устройство;
- операционная система видит плату Meshtastic как новое USB‑устройство /
  последовательный порт после установки драйверов.

## Windows

1. Подключите плату и откройте «Диспетчер устройств».
2. Раскройте раздел «Порты (COM и LPT)».
3. Должен появиться новый COM‑порт, например «Silicon Labs CP210x USB to UART
   (COM5)» или «USB Serial Device (COMx)».

Если нового порта нет:

- убедитесь, что установлен драйвер для вашего чипа (CP210x или CH34x);
- попробуйте другой USB‑порт или кабель.

## Linux

1. Подключите плату к USB.
2. Выполните в терминале:
   ```bash
   lsusb
   ```
3. В списке должно появиться новое USB‑устройство с указанием производителя
   (Silicon Labs, QinHeng/WCH, Adafruit/RAK и т.п.).
4. Дополнительно посмотрите список последовательных портов:
   ```bash
   ls /dev/ttyUSB* /dev/ttyACM* 2>/dev/null
   ```

Если никаких новых устройств не появилось, попробуйте другой USB‑порт или
кабель и проверьте, что драйверы для вашего моста (CP210x или CH34x) установлены.

## macOS

1. Откройте меню ` > Об этом Mac > Подробнее > Отчёт о системе…`.
2. В разделе «USB» найдите свою плату — она должна появиться как устройство с
   упоминанием CP210x, CH9102/CH34x или микроконтроллера.
3. В терминале можно посмотреть список последовательных портов:
   ```bash
   ls /dev/cu.*
   ```
   после подключения платы должен появиться новый `cu.SLAB_USBtoUART`,
   `cu.usbserial-*` или похожее устройство.

Если устройство не видно, проверьте драйверы, перезагрузите Mac и повторите
подключение.

## Если устройство так и не видно

- Замените кабель на заведомо исправный, который поддерживает передачу данных, а не только зарядку.
- Попробуйте подключиться напрямую к порту компьютера, без хаба.
- Переустановите драйверы для нужной платформы:
  - [драйверы для ESP32](./esp32.md)
  - [драйверы для nRF52/RP2040](./nrf52.md)
